Printing Code

created by: Mark Kleback

Printing Code, a 14-week course at ITP, used computational techniques to produce physical prints. For this course, I generated show flyers, logos, and album covers using Processing and openFrameWorks to build each design.

Ice Cream Cone

The above design was the first assignment - make an ice cream cone using only the ellipse(), rect(), and triangle() functions in processing. Only use black (0) and white (255)

Ice Cream Cone on Github

Sharp and Wet

Using only beginShape() and endShape() and vertex functions, create two images. The first is a visual representation of the word "sharp." The second is a representation of the word "wet." Only use black and white.

Sharp and Wet on Github

Waveforms

The object of this assignment was to create an abstract representation of your personality using a specific HSB color scheme.

Waveforms on Github

Typography

For this assignment, we were to design a word by either generating a typeface or modifying an existing font. The word "Analog" here was built using vertex() points. The word "Digital" was generated using the RFont library in Processing.

Generative Fonts on Github

Grids

Using a grid system, I generated a flyer for a show at a music venue. The colors and placements of the items in the grid are random, so every time the code runs, it generates a new flyer.

Grid Flyers on Github

Logo Design

This logo was generated for my band, Mega Yes. Using a Rutt-Etra algorithm in openFrameWorks, I was able to open an audio track and use the volume of each frequency to offset the text. Because the text was three-dimensional, I could also rotate and orient it while the song played. I took snapshots of the logo while the track played to generate this dynamic logo.

Mega Yes Rutt-Etra Visualizer from Mark Kleback on Vimeo.

Mega Yes logo on Github
